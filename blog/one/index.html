Build and deploy a static website with Hugo

September 2, 2017   

Overview

Here is a list of technologies we will be using in this article:

  1. Hugo (Static-site generator)
  2. Github/Github-Pages (Hosting)

Let’s get started!

Setting up Github

Github offers Free hosting for static pages, so we’ll be using that for our workflow and hosting our personal site.

Navigate to github and create the repository that will hold your static files with the name:

[your-github-username].github.io

Don’t clone your repo yet; we’ll get to that soon.

Awesome! Your github is all set and now we can start installing Hugo.

Installing Hugo

For the sake of convenience, I will use the instructions on the Hugo Website.

Brew installation:

brew install hugo

You can see all the available commands with:

hugo --help

Creating your first Hugo site

Make sure you are inside of your personal website directory before continuing.

Run the following command to create your Hugo instance

hugo new site [your website name]

Congrats! You now have Hugo set up.

Adding a Hugo Theme

You can find all the hugo themese Here.

When you have found your theme, you can add the theme in your themes folder as a submodule with the following command:

git submodule add [theme-http-url] themes/[theme-name];

This will help us deploy our hugo site easily later on.

Now that we have our theme, we will need to build our site with the theme. Run the command:

hugo -t [theme-name]

Note: you don’t need to specify theme/[theme-name]. Hugo will search for the specified theme name inside of the theme folder!

Running the local server

Let’s see what our beautiful website looks like! Run the following command:

hugo server -D

This will build all drafted content and start our server.

Open up your favorite browser and navigate to:

localhost:1313

Deploying your Hugo Web Page

After running the server and building your webpage, a public directory will be generated which contains all the necessary files for your website like the index.html, css files, etc.

Github only servers static websites and looks for an index.html to serve to your visiters. Therefore, we need to initialize your main repo inside of the public/ folder.

Go ahead and initialize a git local repo inside of the public/ directory:

git init

Add all the files inside of your local repo and commit your changes

git add . 

git commit -m "Describe your changes"

Now lets set the remote URL of your repo

git remote add origin [remote-repository-URL]

git remote -v

All connected! Let’s push our changes to master

git push -u origin master

Awesome! Our website should be live in 1-2 minutes. go ahead and navigate to [github-username].github.io to see your new website.

Wrapping Up

Now that you have a website you might want to set up a custom DNS. Depending on your DNS Provider, you will need to follow specific instructions through their own tutorials. I’ll post a tutorial on setting up the DNS with namecheap in another article.

Happy learning!
